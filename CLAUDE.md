# TROCCO Pipeline Builder

Build TROCCO data pipelines from natural language using Claude Code.

## Quick Start

```bash
git clone https://github.com/kentayamamoto/trocco-pipeline-builder.git
cd trocco-pipeline-builder
cp .env.example .env.local   # Configure credentials
/setup-pipeline kintone to BigQuery
/setup-pipeline kintone to BigQuery --dry-run
/setup-pipeline kintone to Snowflake
/setup-pipeline google spreadsheets to Snowflake
```

## How It Works

`/setup-pipeline` autonomously executes:

1. Environment check (Terraform, API Key)
2. Source schema retrieval (e.g., kintone Get Form Fields API)
3. Field → column type mapping (per `reference/type-mapping.md`)
4. Terraform HCL generation (per `reference/terraform-patterns.md`)
5. `terraform plan` (present to user)
6. `terraform apply` (user approval required)
7. Test run (TROCCO API)
8. Result report

**No programming.** Markdown prompts + HCL declarations only.

## Supported Connectors

### Sources
kintone, MySQL, PostgreSQL, BigQuery, Salesforce, Google Spreadsheets, S3, GCS, Snowflake, HTTP, Google Analytics 4, Google Ads, SFTP, Databricks, HubSpot

### Destinations
BigQuery, Snowflake*, MySQL, PostgreSQL, Salesforce, Google Spreadsheets, kintone, Databricks, SFTP

*Snowflake: Terraform Provider with REST API fallback.

## Environment Variables

Set in `.env.local` (copy from `.env.example`):

- `TROCCO_API_KEY` — Required
- **kintone source (2 modes):**
  - Mode A: `KINTONE_CONNECTION_ID` — Use existing TROCCO connection (recommended)
  - Mode B: `KINTONE_DOMAIN` / `KINTONE_API_TOKEN` — Create new connection via Terraform
  - `KINTONE_APP_ID` — Always required
- **Google Spreadsheets source (2 modes):**
  - Mode A: `GS_CONNECTION_ID` — Use existing TROCCO connection (recommended)
  - Mode B: `GS_SERVICE_ACCOUNT_JSON_KEY` — Create new connection via Terraform
  - `GS_SPREADSHEET_ID` — Spreadsheet ID from URL (always required)
  - `GS_WORKSHEET_TITLE` — Worksheet name (optional, default: first sheet)
- **BigQuery destination:** `BQ_CONNECTION_ID` / `BQ_DATASET` / `BQ_TABLE`
- **Snowflake destination (2 modes):**
  - Mode A: `SNOWFLAKE_CONNECTION_ID` — Use existing TROCCO connection
  - Mode B: `SNOWFLAKE_HOST` / `SNOWFLAKE_USER` / `SNOWFLAKE_PASSWORD` or `SNOWFLAKE_PRIVATE_KEY` / `SNOWFLAKE_WAREHOUSE` / `SNOWFLAKE_DATABASE` / `SNOWFLAKE_AUTH_METHOD` (optional, default: `user_password`)
  - `SNOWFLAKE_ROLE` — Required (no default; specify the role created for TROCCO)

## Security

- Credentials in `.env.local` only (gitignored)
- Sensitive Terraform vars via `TF_VAR_xxx` env injection
- **Important:** Never write sensitive values (even as empty strings) in `terraform.tfvars`. Terraform's variable precedence is `tfvars > env vars`, so `trocco_api_key = ""` in tfvars would override `TF_VAR_trocco_api_key`.
- `terraform apply` requires explicit user approval
- kintone record data is never fetched (field definitions only)
