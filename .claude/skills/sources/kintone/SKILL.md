# kintone ソースSkill

オーケストレーター (`setup-pipeline.md`) から Read で読み込まれ、kintone ソース固有の処理を実行する。

## 必要環境変数

| 変数 | モードA | モードB | 説明 |
|------|---------|---------|------|
| `KINTONE_CONNECTION_ID` | 必須 | 不要 | 既存TROCCO kintone接続ID |
| `KINTONE_DOMAIN` | 不要 | 必須 | kintoneサブドメイン (例: `example.cybozu.com`) |
| `KINTONE_API_TOKEN` | 不要 | 必須 | kintone APIトークン（閲覧権限） |
| `KINTONE_APP_ID` | 必須 | 必須 | kintoneアプリID |

**モード判定:**
- `KINTONE_CONNECTION_ID` が設定済み → **モードA**（既存接続参照、推奨）
- `KINTONE_CONNECTION_ID` が空 → **モードB**（Terraformで新規作成）。`KINTONE_DOMAIN` + `KINTONE_API_TOKEN` が必須。

## Step 1: スキーマ取得

kintone Get Form Fields API でフィールド定義を取得する。

```bash
set -a && source .env.local && set +a
RESPONSE=$(curl -s "https://${KINTONE_DOMAIN}/k/v1/app/form/fields.json" \
  -G -d "app=${KINTONE_APP_ID}" \
  -H "X-Cybozu-API-Token: ${KINTONE_API_TOKEN}" \
  -w "\nHTTP_STATUS:%{http_code}")

HTTP_CODE=$(echo "$RESPONSE" | tail -1 | sed 's/HTTP_STATUS://')
BODY=$(echo "$RESPONSE" | sed '$d')
echo "$BODY" > /tmp/kintone-fields-response.json
echo "HTTP Status: $HTTP_CODE"
```

### HTTPステータス確認

- 200: 正常 → Step 2 へ
- 401/403: 認証エラー → 「KINTONE_API_TOKENが無効、またはアプリへの閲覧権限がありません」と案内して停止
- 404: アプリ不存在 → 「KINTONE_APP_IDが正しいか確認してください」と案内して停止
- その他: エラー内容を表示して停止

### API呼び出しの注意

- **GET リクエストに Content-Type ヘッダを付けない。** kintone API は GET リクエストに `Content-Type: application/json` を付けると HTTP 400 エラーを返す。
- モードA（既存接続ID参照）で `KINTONE_DOMAIN` / `KINTONE_API_TOKEN` が未設定の場合は、スキーマ取得をスキップしてユーザーにフィールド情報を手動入力してもらう。

## Step 2: 型変換

**reference/type-mapping.md** の kintone セクションに従い、フィールド情報を変換する。

```bash
jq -r '
  .properties | to_entries[]
  | select(IN(.value.type; "FILE","STATUS_ASSIGNEE","CATEGORY","REFERENCE_TABLE") | not)
  | [.key, .value.type, .value.label] | @tsv
' /tmp/kintone-fields-response.json | sort
```

> **注意:** jqフィルタはシングルクォートで囲む。不等号比較はINパターン（select + not）を使用。jqはファイル引数で呼ぶ。

type-mapping.md の変換ルールを適用し:
1. `input_option_columns` 配列を生成（kintoneフィールドコード + TROCCOカラムタイプ）
2. `filter_columns` 配列を生成（英語カラム名 + src + タイプ + format）

日本語フィールドコードの英語変換はあなた（Claude）の推論で実行してください。
例: `顧客名` → `customer_name`, `売上金額` → `sales_amount`

## Step 3 (src): 接続確認

### モードA: 既存接続ID参照

`KINTONE_CONNECTION_ID` が設定済み → そのIDを使用。API呼び出しスキップ。

### モードB: Terraform新規作成

`KINTONE_CONNECTION_ID` が空 → API一覧を試行:

```bash
set -a && source .env.local && set +a
if [ -n "$KINTONE_CONNECTION_ID" ]; then
  echo "KINTONE_CONNECTION_ID=${KINTONE_CONNECTION_ID} (設定済み → このIDを使用)"
else
  RESULT=$(curl -s -w "\nHTTP_STATUS:%{http_code}" \
    "https://trocco.io/api/connections/kintone" \
    -H "Authorization: Token ${TROCCO_API_KEY}")
  HTTP_CODE=$(echo "$RESULT" | tail -1 | sed 's/HTTP_STATUS://')
  BODY=$(echo "$RESULT" | sed '$d')

  if [ "$HTTP_CODE" = "200" ]; then
    echo "$BODY" | jq '.items[] | {id, name}' 2>/dev/null || echo "既存接続なし"
  elif [ "$HTTP_CODE" = "401" ] || [ "$HTTP_CODE" = "403" ]; then
    echo "API HTTP ${HTTP_CODE}: 接続一覧APIへのアクセス権限がありません"
    echo "→ TROCCO管理画面 > 接続情報 でIDを確認し .env.local に設定してください"
  else
    echo "API HTTP ${HTTP_CODE}: ${BODY}"
  fi
fi
```

- API成功 + 既存接続あり → ユーザーにID設定を案内
- API失敗（401/403） → TROCCO管理画面での手動確認を案内
- 接続なし → Terraformで新規作成。`KINTONE_DOMAIN` と `KINTONE_API_TOKEN` が必須。

## HCL情報

### input_option 構造

```
input_option_type = "kintone"
input_option = {
  kintone_input_option = {
    kintone_connection_id = local.kintone_connection_id
    app_id                = var.kintone_app_id
    guest_space_id        = var.kintone_guest_space_id
    expand_subtable       = var.kintone_expand_subtable

    input_option_columns = var.input_columns
  }
}
```

### connection resource (モードB時)

```hcl
resource "trocco_connection" "kintone_source" {
  count = var.kintone_connection_id == null ? 1 : 0

  name            = var.kintone_connection_name
  connection_type = "kintone"
  description     = "kintone ${var.kintone_app_name} - auto-generated by TROCCO Pipeline Builder"

  domain       = var.kintone_domain
  login_method = "token"
  token        = var.kintone_api_token
}

locals {
  kintone_connection_id = (
    var.kintone_connection_id != null
    ? var.kintone_connection_id
    : trocco_connection.kintone_source[0].id
  )
}
```
