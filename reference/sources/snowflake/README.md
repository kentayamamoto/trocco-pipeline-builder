# Snowflake Source

## Overview

Snowflake をデータソースとして使用し、TROCCO の `snowflake_input_option` でデータを転送する。
テーブル指定またはカスタム SQL クエリの両方をサポート。

**注意:** Snowflake はデスティネーションとしても実装済み（`SNOWFLAKE_*` プレフィックス）。
ソース側は `SNOWFLAKE_SRC_*` プレフィックスを使用し、同一パイプラインでの併用時の衝突を回避する。

## Required Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `SNOWFLAKE_SRC_HOST` | Account URL | `xxx.snowflakecomputing.com` |
| `SNOWFLAKE_SRC_USER` | Username | `TROCCO_USER` |
| `SNOWFLAKE_SRC_AUTH_METHOD` | Auth method: `user_password` or `key_pair` | `user_password` |
| `SNOWFLAKE_SRC_PASSWORD` | Password (user_password auth) | (sensitive) |
| `SNOWFLAKE_SRC_PRIVATE_KEY` | Private key (key_pair auth) | (sensitive) |
| `SNOWFLAKE_SRC_DATABASE` | Database name | `DEMO_DB` |
| `SNOWFLAKE_SRC_SCHEMA` | Schema name | `PUBLIC` |
| `SNOWFLAKE_SRC_QUERY` | SQL query to extract data | `SELECT * FROM users` |
| `SNOWFLAKE_SRC_WAREHOUSE` | Warehouse name | `COMPUTE_WH` |
| `SNOWFLAKE_SRC_ROLE` | Role name (required) | `TROCCO_ROLE` |

## Connection Method

### Mode A: 既存接続ID参照（推奨）

TROCCO管理画面で事前に Snowflake 接続を作成し、`SNOWFLAKE_SRC_CONNECTION_ID` を設定。

### Mode B: Terraform 新規作成

`SNOWFLAKE_SRC_CONNECTION_ID` が未設定の場合、以下の環境変数で新規接続を作成:
- `SNOWFLAKE_SRC_HOST` — Snowflake アカウントのホスト名
- `SNOWFLAKE_SRC_USER` — ユーザー名
- `SNOWFLAKE_SRC_AUTH_METHOD` — `user_password`（デフォルト）または `key_pair`
- `SNOWFLAKE_SRC_PASSWORD` — パスワード（user_password の場合）
- `SNOWFLAKE_SRC_PRIVATE_KEY` — 秘密鍵（key_pair の場合）

## Private Key Format (key_pair auth)

`.env.local` に秘密鍵を設定する際は、改行を `\n` リテラルで記載する:
```
SNOWFLAKE_SRC_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----"
```

Terraform 注入時に `printf '%b'` で実際の改行に自動変換される。
`echo -e` ではなく `printf '%b'` を使用（POSIX互換性が高い）。

## Schema Retrieval

### Snowflake SQL REST API

Snowflake SQL REST API (`POST https://{account}/api/v2/statements`) で `INFORMATION_SCHEMA.COLUMNS` をクエリし、カラム定義を取得する。

```bash
# JWT生成（key_pair認証の場合）
# openssl を使用して JWT を生成

# SQL REST API 呼び出し
ACCOUNT_HOST="${SNOWFLAKE_SRC_HOST}"
curl -s -X POST "https://${ACCOUNT_HOST}/api/v2/statements" \
  -H "Authorization: Bearer ${JWT_TOKEN}" \
  -H "Content-Type: application/json" \
  -H "X-Snowflake-Authorization-Token-Type: KEYPAIR_JWT" \
  -d '{
    "statement": "SELECT COLUMN_NAME, DATA_TYPE, NUMERIC_PRECISION, NUMERIC_SCALE, IS_NULLABLE FROM '"${SNOWFLAKE_SRC_DATABASE}"'.INFORMATION_SCHEMA.COLUMNS WHERE TABLE_SCHEMA = '"'"''"${SNOWFLAKE_SRC_SCHEMA}"''"'"' AND TABLE_NAME = '"'"''"${SNOWFLAKE_SRC_TABLE}"''"'"' ORDER BY ORDINAL_POSITION",
    "timeout": 60,
    "database": "'"${SNOWFLAKE_SRC_DATABASE}"'",
    "schema": "INFORMATION_SCHEMA",
    "warehouse": "'"${SNOWFLAKE_SRC_WAREHOUSE}"'",
    "role": "'"${SNOWFLAKE_SRC_ROLE}"'"
  }'
```

### 認証方式

- **key_pair**: openssl で JWT を生成し、`X-Snowflake-Authorization-Token-Type: KEYPAIR_JWT` で認証
- **user_password**: snowsql CLI を使用するか、手動入力にフォールバック

### フォールバック

REST API でのスキーマ取得に失敗した場合、ユーザーにカラム情報を手動入力してもらう。

### Response Format

```json
{
  "data": [
    ["ID", "NUMBER", "38", "0", "NO"],
    ["NAME", "VARCHAR", null, null, "YES"],
    ["CREATED_AT", "TIMESTAMP_NTZ", null, null, "YES"]
  ]
}
```

## Terraform Configuration

### Connection

```hcl
resource "trocco_connection" "snowflake_source" {
  count = var.snowflake_src_connection_id == null ? 1 : 0

  name            = var.snowflake_src_connection_name
  connection_type = "snowflake"
  description     = "Snowflake ${var.snowflake_src_database}.${var.snowflake_src_schema} (source) - auto-generated by TROCCO Pipeline Builder"

  host        = var.snowflake_src_host
  user_name   = var.snowflake_src_user
  auth_method = var.snowflake_src_auth_method
  password    = var.snowflake_src_auth_method == "user_password" ? var.snowflake_src_password : null
  private_key = var.snowflake_src_auth_method == "key_pair" ? var.snowflake_src_private_key : null
  role        = var.snowflake_src_role
}
```

### Input Option

```hcl
input_option_type = "snowflake"
input_option = {
  snowflake_input_option = {
    snowflake_connection_id = local.snowflake_src_connection_id
    database                = var.snowflake_src_database
    schema                  = var.snowflake_src_schema
    query                   = var.snowflake_src_query
    warehouse               = var.snowflake_src_warehouse

    input_option_columns = var.input_columns
  }
}
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `snowflake_connection_id` | number | Yes | TROCCO Snowflake connection ID |
| `database` | string | Yes | Source database |
| `schema` | string | No | Source schema（デフォルト: Provider側で自動設定） |
| `query` | string | Yes | SQL query to extract data |
| `warehouse` | string | Yes | Snowflake warehouse |
| `input_option_columns` | list | Yes | カラム定義（name + type） |

## Query Examples

テーブル全体を取得:
```
SNOWFLAKE_SRC_QUERY="SELECT * FROM my_table"
```

条件付きクエリ:
```
SNOWFLAKE_SRC_QUERY="SELECT id, name, email FROM users WHERE active = true"
```

## Required Snowflake Permissions

```sql
-- データベース・スキーマへのアクセス
GRANT USAGE ON DATABASE {db} TO ROLE {role};
GRANT USAGE ON SCHEMA {db}.{schema} TO ROLE {role};

-- テーブルの SELECT 権限
GRANT SELECT ON ALL TABLES IN SCHEMA {db}.{schema} TO ROLE {role};
GRANT SELECT ON FUTURE TABLES IN SCHEMA {db}.{schema} TO ROLE {role};

-- ウェアハウスの利用
GRANT USAGE ON WAREHOUSE {wh} TO ROLE {role};
```

## Notes

- **プレフィックス規則:** ソースは `SNOWFLAKE_SRC_*`、デスティネーションは `SNOWFLAKE_*`（既存維持）
- Snowflake Free Trial (30 days, $400 credits): https://signup.snowflake.com/
- Use `XSMALL` warehouse with `AUTO_SUSPEND=60` for cost efficiency
- Provider は常に `query`（SQL文）を要求する。テーブル全体を取得する場合は `SELECT * FROM table_name` を指定
- `input_option_columns`（name + type）を使用。Salesforce の `columns` とは異なるので注意
