# Amazon S3 Source

## Overview

Amazon S3 をソースとしてファイルデータを転送する。
TROCCO の `s3` input_option でS3バケットのファイルを読み取る。

S3 はファイルベースのソースであり、複数のファイルパーサー（CSV/JSONL/JSONPath/Parquet/Excel/LTSV/XML）をサポートする。
また、AWS認証方式として IAM User と AssumeRole の2種類をサポートする。

S3 接続は2つのモードをサポート:
- **Mode A（推奨）:** TROCCO UI で事前に作成した接続IDを参照
- **Mode B:** Terraform で `trocco_connection` リソースを新規作成（IAM User or AssumeRole認証）

## Required Environment Variables

| Variable | Mode A | Mode B | Description | Example |
|----------|--------|--------|-------------|---------|
| `S3_CONNECTION_ID` | 必須 | 不要 | TROCCO S3接続ID | `123` |
| `S3_AWS_AUTH_TYPE` | 不要 | 必須 | `iam_user` or `assume_role` | `iam_user` |
| `S3_AWS_ACCESS_KEY_ID` | 不要 | 条件付 | IAM User時のアクセスキー | `AKIAIOSFODNN7EXAMPLE` |
| `S3_AWS_SECRET_ACCESS_KEY` | 不要 | 条件付 | IAM User時のシークレットキー | |
| `S3_AWS_ACCOUNT_ID` | 不要 | 条件付 | AssumeRole時のAWSアカウントID | `123456789012` |
| `S3_AWS_ROLE_NAME` | 不要 | 条件付 | AssumeRole時のIAMロール名 | `trocco-s3-role` |
| `S3_BUCKET` | 必須 | 必須 | S3バケット名 | `my-data-bucket` |
| `S3_PATH_PREFIX` | 必須 | 必須 | パスプレフィックス | `data/csv/` |
| `S3_REGION` | 任意 | 任意 | AWSリージョン（デフォルト: ap-northeast-1） | `ap-northeast-1` |
| `S3_FILE_FORMAT` | 必須 | 必須 | ファイルフォーマット | `csv` |

### サポートするファイルフォーマット

| フォーマット | `S3_FILE_FORMAT` 値 | スキーマ自動取得 |
|-------------|-------------------|----------------|
| CSV | `csv` | ヘッダー行 + サンプルデータ |
| JSONL | `jsonl` | キー名 + 値型 |
| JSONPath | `jsonpath` | 手動入力（root式が必要） |
| Parquet | `parquet` | 手動入力（Python非依存のため） |
| Excel | `excel` | 手動入力（Python非依存のため） |
| LTSV | `ltsv` | ラベル名 |
| XML | `xml` | 手動入力 |

## Connection Method

### Mode A: 既存接続ID参照（推奨）

TROCCO 管理画面で以下の手順で接続を作成し、接続IDを `.env.local` の `S3_CONNECTION_ID` に設定する:

1. TROCCO 管理画面 > 接続情報 > 新規作成
2. 接続タイプ: Amazon S3
3. 認証方法: IAM User または AssumeRole
4. 認証完了後、接続IDを `.env.local` に設定

#### 接続ID確認

```bash
source .env.local
curl -s "https://trocco.io/api/connections/s3" \
  -H "Authorization: Token ${TROCCO_API_KEY}" \
  | jq '.items[] | {id, name}'
```

### Mode B: Terraform で新規作成

IAM User または AssumeRole の認証情報を使って Terraform で接続を自動作成する。

#### IAM User 認証

1. AWS IAM でS3読み取り権限を持つユーザーを作成
2. アクセスキーとシークレットキーを取得
3. `.env.local` に以下を設定:
   - `S3_AWS_AUTH_TYPE="iam_user"`
   - `S3_AWS_ACCESS_KEY_ID` と `S3_AWS_SECRET_ACCESS_KEY`

#### AssumeRole 認証

1. AWS IAM でTROCCOからのAssumeRoleを許可するロールを作成
2. TROCCOのAWSアカウントからの信頼関係を設定
3. `.env.local` に以下を設定:
   - `S3_AWS_AUTH_TYPE="assume_role"`
   - `S3_AWS_ACCOUNT_ID` と `S3_AWS_ROLE_NAME`

#### Terraform Configuration (Mode B)

**⚠ Provider 互換性注意:** `aws_assume_role = ... : null` の三項演算子パターンは Provider エラーを引き起こす。`aws_auth_type` に応じて使用する認証ブロックのみ生成すること。

**IAM User 認証の場合:**
```hcl
resource "trocco_connection" "s3_source" {
  count = var.s3_connection_id == null ? 1 : 0

  name            = var.s3_connection_name
  connection_type = "s3"
  description     = "S3 ${var.s3_bucket} - auto-generated by TROCCO Pipeline Builder"

  aws_auth_type = "iam_user"

  aws_iam_user = {
    access_key_id     = var.s3_aws_access_key_id
    secret_access_key = var.s3_aws_secret_access_key
  }
}
```

**AssumeRole 認証の場合:**
```hcl
resource "trocco_connection" "s3_source" {
  count = var.s3_connection_id == null ? 1 : 0

  name            = var.s3_connection_name
  connection_type = "s3"
  description     = "S3 ${var.s3_bucket} - auto-generated by TROCCO Pipeline Builder"

  aws_auth_type = "assume_role"

  aws_assume_role = {
    account_id = var.s3_aws_account_id
    role_name  = var.s3_aws_role_name
  }
}
```

```hcl
locals {
  s3_connection_id = (
    var.s3_connection_id != null
    ? var.s3_connection_id
    : trocco_connection.s3_source[0].id
  )
}
```

## Schema Retrieval

ファイルフォーマットによってスキーマ取得方法が異なる。

### 方法1: AWS CLI でサンプルファイル取得（CSV/JSONL/LTSV）

AWS CLI がインストール済みで認証が設定されている場合:

```bash
set -a && source .env.local && set +a

# AWS CLIが利用可能か確認
if command -v aws &>/dev/null; then
  # 先頭ファイルを取得
  FIRST_FILE=$(aws s3 ls "s3://${S3_BUCKET}/${S3_PATH_PREFIX}" --region "${S3_REGION:-ap-northeast-1}" \
    | grep -v '/$' | head -1 | awk '{print $NF}')

  if [ -n "$FIRST_FILE" ]; then
    aws s3 cp "s3://${S3_BUCKET}/${S3_PATH_PREFIX}${FIRST_FILE}" /tmp/s3-sample-file \
      --region "${S3_REGION:-ap-northeast-1}"
    echo "Downloaded: ${FIRST_FILE}"
  fi
fi
```

#### CSV の場合

```bash
# ヘッダー行を取得
head -1 /tmp/s3-sample-file
# サンプルデータを取得（2-6行目）
sed -n '2,6p' /tmp/s3-sample-file
```

#### JSONL の場合

```bash
# キー名と型を取得
head -1 /tmp/s3-sample-file | jq 'to_entries[] | {key, type: (.value | type)}'
```

#### LTSV の場合

```bash
# ラベル名を取得
head -1 /tmp/s3-sample-file | tr '\t' '\n' | cut -d: -f1
```

### 方法2: ユーザー手動入力（フォールバック）

AWS CLI 未インストール、認証未設定、または Parquet/Excel/JSONPath/XML フォーマットの場合:

確認する情報:
1. カラム名一覧（例: id, name, email, amount, created_at）
2. 各カラムのデータ型（整数、小数、日付、文字列など）
3. CSV の場合: 区切り文字、ヘッダー行の有無
4. JSONPath の場合: root式（JSONPath表現）

## Terraform Configuration

### s3_input_option Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `s3_connection_id` | number | Yes | - | TROCCO S3接続ID |
| `bucket` | string | Yes | - | S3バケット名 |
| `region` | string | No | - | AWSリージョン |
| `path_prefix` | string | No | - | パスプレフィックス |
| `path_match_pattern` | string | No | - | パスマッチパターン（正規表現） |
| `is_skip_header_line` | bool | No | `false` | ヘッダー行スキップ |
| `incremental_loading_enabled` | bool | No | `false` | 差分ロード有効化 |
| `stop_when_file_not_found` | bool | No | `false` | ファイル未検出時に停止 |
| `decompression_type` | string | No | - | 解凍方式（gzip等） |
| `decoder` | object | **Yes** | - | デコーダー設定（⚠ 省略すると "Provider produced inconsistent result" エラー。`{ match_name = "" }` を必ず指定） |

### Input Option

**重要設計方針:** パーサー選択にTerraformの条件分岐は使わない。ファイル形式はHCL生成時に確定しているため、選択されたパーサーのブロックのみ生成する。

```hcl
input_option_type = "s3"
input_option = {
  s3_input_option = {
    s3_connection_id = local.s3_connection_id
    bucket           = var.s3_bucket
    region           = var.s3_region
    path_prefix      = var.s3_path_prefix

    # ⚠ decoder は必須。省略すると "Provider produced inconsistent result" エラー
    decoder = {
      match_name = ""
    }

    # パーサーは1つだけ生成（ファイル形式に応じて選択）
    csv_parser = {          # ← CSV の場合のみ
      columns           = var.input_columns
      delimiter         = var.s3_csv_delimiter
      skip_header_lines = var.s3_csv_skip_header_lines
      default_time_zone = var.s3_default_time_zone
    }
  }
}
```

#### csv_parser

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `columns` | list | Yes | - | カラム定義 |

| `escape` | string | No | `"\\"` | エスケープ文字 |

#### excel_parser

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `columns` | list | Yes | - | カラム定義（`formula_handling` フィールド必須） |
| `sheet_name` | string | Yes | - | シート名（必須） |

| `default_time_zone` | string | No | - | デフォルトタイムゾーン |

**Excel columns 構造（formula_handling 付き）:**
```hcl
columns = [
  { name = "id",     type = "long",   formula_handling = "cashed_value" },
  { name = "name",   type = "string", formula_handling = "cashed_value" },
  { name = "amount", type = "double", formula_handling = "evaluate" },
]
```
- `formula_handling`: 各カラムで必須。`"cashed_value"`（キャッシュ値使用、推奨）or `"evaluate"`（数式を評価）
- **注意:** Provider の命名は `cashed_value`（typo）であり、`cached_value` ではない

#### ltsv_parser

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `columns` | list | Yes | - | カラム定義 |

#### xml_parser

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `columns` | list | Yes | - | カラム定義 |

### columns 共通構造

```hcl
columns = [
  { name = "id",         type = "long" },
  { name = "name",       type = "string" },
  { name = "amount",     type = "double" },
  { name = "created_at", type = "timestamp", format = "%Y-%m-%d %H:%M:%S" },
]
```

## Type Mapping

See `reference/type-mapping.md` の「Amazon S3 / ファイルベースソース」セクションを参照。

ファイルフォーマットによって型推論方法が異なる:
- CSV/LTSV: サンプルデータからの推論（Google Spreadsheetsと同じルール）
- JSONL: JSONネイティブ型マッピング
- Parquet: Arrow型からの直接マッピング
- Excel: セルの書式情報から推論

## Notes

- Mode A: TROCCO UI で事前作成（IAM User or AssumeRole認証）
- Mode B: Terraform で新規作成（IAM User or AssumeRole認証）
- S3バケットのファイルを読み取り専用（書き込みは行わない）
- `path_prefix` と `path_match_pattern` で対象ファイルを絞り込む
- パーサーは1つだけ選択（ファイル形式に応じてHCL生成時に確定）
- 複数のファイルフォーマットを同一ジョブで混在させることはできない
