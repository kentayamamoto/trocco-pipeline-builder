# Google Spreadsheets Source

## Overview

Google Spreadsheets をソースとしてデータを転送する。
TROCCO の `google_spreadsheets` input_option でスプレッドシートのデータを読み取る。

Google Spreadsheets 接続は2つのモードをサポート:
- **Mode A（推奨）:** TROCCO UI で事前に作成した接続IDを参照
- **Mode B:** Terraform で `trocco_connection` リソースを新規作成（サービスアカウント認証）

## Required Environment Variables

| Variable | Mode A | Mode B | Description | Example |
|----------|--------|--------|-------------|---------|
| `GS_CONNECTION_ID` | 必須 | 不要 | TROCCO GS接続ID | `123` |
| `GS_SERVICE_ACCOUNT_JSON_KEY` | 不要 | 必須 | サービスアカウントJSONキー | `{"type":"service_account",...}` |
| `GS_SPREADSHEET_ID` | 必須 | 必須 | スプレッドシートID（URLから取得） | `1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms` |
| `GS_WORKSHEET_TITLE` | 任意 | 任意 | ワークシート名（デフォルト: 先頭シート） | `Sheet1` |

### スプレッドシートID の取得方法

スプレッドシートのURLから `spreadsheets/d/` と `/edit` の間の文字列がID:
```
https://docs.google.com/spreadsheets/d/{SPREADSHEET_ID}/edit
```

## Connection Method

### Mode A: 既存接続ID参照（推奨）

TROCCO 管理画面で以下の手順で接続を作成し、接続IDを `.env.local` の `GS_CONNECTION_ID` に設定する:

1. TROCCO 管理画面 > 接続情報 > 新規作成
2. 接続タイプ: Google Spreadsheets
3. 認証方法: Google アカウント（OAuth）またはサービスアカウント
4. 認証完了後、接続IDを `.env.local` に設定

#### 接続ID確認

```bash
source .env.local
curl -s "https://trocco.io/api/connections/google_spreadsheets" \
  -H "Authorization: Token ${TROCCO_API_KEY}" \
  | jq '.items[] | {id, name}'
```

### Mode B: Terraform で新規作成（サービスアカウント認証）

サービスアカウントのJSONキーを使って Terraform で接続を自動作成する。
TROCCO Terraform Provider v0.8.0 以降で `connection_type = "google_spreadsheets"` + `service_account_json_key` がサポートされている。

1. Google Cloud Console でサービスアカウントを作成
2. サービスアカウントにスプレッドシートの閲覧権限を付与
3. JSONキーをダウンロードし、`.env.local` の `GS_SERVICE_ACCOUNT_JSON_KEY` に設定
4. `GS_CONNECTION_ID` を空にしておく → Terraform が自動的に接続を作成

#### Terraform Configuration (Mode B)

```hcl
resource "trocco_connection" "google_spreadsheets_source" {
  count = var.gs_connection_id == null ? 1 : 0

  name            = var.gs_connection_name
  connection_type = "google_spreadsheets"
  description     = "Google Spreadsheets - auto-generated by TROCCO Pipeline Builder"

  service_account_json_key = var.gs_service_account_json_key
}

locals {
  gs_connection_id = (
    var.gs_connection_id != null
    ? var.gs_connection_id
    : trocco_connection.google_spreadsheets_source[0].id
  )
}
```

## Schema Retrieval

Google Spreadsheets にはフォーマルなスキーマ定義がないため、以下の方法でカラム情報を取得する。

### 方法1: Google Sheets API v4 によるヘッダー行取得

サービスアカウントのキーファイルが利用可能な場合:

```bash
set -a && source .env.local && set +a
if [ -n "$GS_SERVICE_ACCOUNT_JSON_KEY" ]; then
  echo "$GS_SERVICE_ACCOUNT_JSON_KEY" > /tmp/credential.json
  gcloud auth activate-service-account --key-file=/tmp/credential.json 2>/dev/null
  ACCESS_TOKEN=$(gcloud auth print-access-token \
    --scopes="https://www.googleapis.com/auth/spreadsheets.readonly")
  rm -f /tmp/credential.json

  WORKSHEET="${GS_WORKSHEET_TITLE:-Sheet1}"
  # URLエンコード（日本語ワークシート名対応、!も含めてエンコード）
  ENCODED_RANGE=$(WORKSHEET="$WORKSHEET" python3 << 'PYEOF'
import urllib.parse, os
w = os.environ['WORKSHEET']
print(urllib.parse.quote(w + chr(33) + '1:3'))
PYEOF
)
  RESPONSE=$(curl -s -w "\nHTTP_STATUS:%{http_code}" \
    "https://sheets.googleapis.com/v4/spreadsheets/${GS_SPREADSHEET_ID}/values/${ENCODED_RANGE}" \
    -H "Authorization: Bearer ${ACCESS_TOKEN}")
  HTTP_CODE=$(echo "$RESPONSE" | tail -1 | sed 's/HTTP_STATUS://')
  BODY=$(echo "$RESPONSE" | sed '$d')
  echo "$BODY" > /tmp/gs-header-response.json
  echo "HTTP Status: $HTTP_CODE"
fi
```

### 方法2: ユーザー手動入力

API が利用できない場合は、ユーザーにカラム名と型を手動で入力してもらう。
Claude Code がスプレッドシートの構造についてユーザーに質問し、情報を収集する。

確認する情報:
1. ヘッダー行のカラム名一覧
2. 各カラムのデータ型（整数、小数、日付、文字列など）
3. データ開始行（通常はヘッダーの次の行 = 2）
4. 開始列（通常は "A"）

## Terraform Configuration

### Input Option

```hcl
input_option_type = "google_spreadsheets"
input_option = {
  google_spreadsheets_input_option = {
    google_spreadsheets_connection_id = local.gs_connection_id
    spreadsheets_url                  = "https://docs.google.com/spreadsheets/d/${var.gs_spreadsheet_id}/edit"
    worksheet_title                   = var.gs_worksheet_title
    start_row                         = var.gs_start_row
    start_column                      = var.gs_start_column
    default_time_zone                 = var.gs_default_time_zone
    null_string                       = var.gs_null_string

    input_option_columns = var.input_columns
  }
}
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `google_spreadsheets_connection_id` | number | Yes | - | TROCCO GS接続ID |
| `spreadsheets_url` | string | Yes | - | スプレッドシートの完全URL |
| `worksheet_title` | string | Yes | - | ワークシート名 |
| `start_row` | number | Yes | `2` | データ開始行（ヘッダー=1なら通常2） |
| `start_column` | string | Yes | `"A"` | 開始列 |
| `default_time_zone` | string | Yes | `"Asia/Tokyo"` | タイムゾーン |
| `input_option_columns` | list | Yes | - | カラム定義 |
| `null_string` | string | No | `""` | NULL扱いする文字列 |

## Type Mapping

See `reference/sources/google_spreadsheets/type-mapping.md` の「Google Spreadsheets」セクションを参照。

Google Spreadsheets にはフォーマルな型定義がないため、
サンプルデータからの推論で型を決定する。デフォルトは `string`。

## Notes

- Mode A: TROCCO UI で事前作成（OAuth またはサービスアカウント認証）
- Mode B: Terraform で新規作成（サービスアカウント認証のみ）
- スプレッドシートのデータのみ読み取り（書き込みは行わない）
- 大量データ（数万行以上）の場合はパフォーマンスに注意
- `spreadsheets_url` は `GS_SPREADSHEET_ID` から自動構築される
